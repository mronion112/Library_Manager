📘 BÀI TẬP LỚN – LẬP TRÌNH JAVA CƠ BẢN & OOP

Đề tài: Ứng dụng Quản lý Thư viện Mini

1. Mục tiêu

Ôn tập Java cơ bản (kiểu dữ liệu, điều kiện, vòng lặp, collection).
Vận dụng OOP (class, object, kế thừa, interface, encapsulation).
Tổ chức mã nguồn thành các lớp rõ ràng.
Thực hành quản lý dữ liệu với ArrayList.
Biết cách đọc/ghi dữ liệu ra file.
Làm quen với thiết kế chương trình có menu console.

2. Mô tả bài toán

Xây dựng ứng dụng console quản lý thư viện mini:
Quản lý sách (mỗi đầu sách có số lượng).
Quản lý người dùng.
Người dùng có thể mượn và trả sách.
Mỗi người dùng có danh sách các sách đã mượn.
Dữ liệu được lưu trữ ra file khi thoát và nạp lại khi khởi động.

3. Yêu cầu chức năng
3.1. Quản lý sách

Mỗi Book có thông tin:
id (mã số duy nhất)
title (tên sách)
author (tác giả)
year (năm xuất bản)
quantity (số lượng còn lại trong thư viện)

Chức năng:

Thêm mới sách.
Xoá sách theo id.
Cập nhật thông tin hoặc số lượng sách.
Tìm kiếm sách theo tên hoặc tác giả.
Hiển thị danh sách sách hiện có.

3.2. Quản lý người dùng

Mỗi User có thông tin:
id
cccd
name
birthdate
address

Danh sách sách đã mượn (list BorrowRecord).

Chức năng:
Thêm người dùng mới.
Hiển thị danh sách người dùng và các sách họ đang mượn.

3.3. Mượn và trả sách

Khi mượn:

Nhập id người dùng, id sách và số lượng.
Nếu sách còn đủ số lượng thì cho mượn.
Giảm số lượng sách trong thư viện.
Ghi nhận sách đã mượn vào danh sách của người dùng.

Khi trả:

Nhập id người dùng, id sách và số lượng.
Tăng số lượng sách trong thư viện.
Giảm số lượng sách trong danh sách mượn của người dùng.
Không cho trả quá số lượng đã mượn.

3.4. Lưu và nạp dữ liệu từ file
File books.txt lưu danh sách sách:
1,Harry Potter,J.K.Rowling,1997,5
2,Lập Trình Java,Nguyen Van A,2020,10

File users.txt lưu danh sách người dùng:
1,033012112233,Nguyen Van B,01/01/2000,Hà Nội
2,033012112233,Tran Thi C,01/01/2000,Hà Nội


File borrow_records.txt (nâng cao, khuyến khích):

userId=1,bookId=2,quantity=2
userId=1,bookId=1,quantity=1


Khi chạy chương trình: đọc file để khởi tạo dữ liệu.
Khi thoát: ghi lại toàn bộ dữ liệu ra file.

3.5. Menu chương trình
================= LIBRARY MENU =================
1. Thêm sách

3. Cập nhật thông tin sách
4. Tìm kiếm sách

7. Hiển thị danh sách người dùng + sách đã mượn

9. Trả sách

4. Gợi ý thiết kế class
class User {
    private int id;
    private String name;
    private List<BorrowRecord> borrowedBooks;
}

class Book {
    private int id;
    private String title;
    private String author;
    private int year;
    private int quantity;
    public boolean isAvailable() { return quantity > 0; }
}

class BorrowRecord {
    private Book book;
    private int quantity;
}

class Library {
    private List<Book> books;
    private List<User> users;
    // các hàm quản lý sách, người dùng, mượn trả
}
